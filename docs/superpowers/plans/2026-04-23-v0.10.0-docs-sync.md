# Nous Tool Gateway

The Nous Tool Gateway gives paid [Nous Portal](https://portal.nousresearch.com) subscribers automatic access to managed cloud tools — web search, image generation, text-to-speech, and browser automation — through their existing subscription, with no additional API keys.

**Available since:** v0.10.0 (v2026.4.16)

---

## Who Gets Access

The Tool Gateway is available to Nous Portal subscribers who are **not** on the free tier. Free-tier accounts are excluded.

When you run `hermes model` and select Nous Portal, Hermes authenticates via OAuth and stores your credentials in `~/.hermes/auth.json`. The gateway layer reads these credentials automatically — no additional configuration is required to enable access.

---

## Covered Tools

| Tool | Vendor | Gateway endpoint |
|------|--------|-----------------|
| Web search & extraction | Firecrawl | `firecrawl-gateway.nousresearch.com` |
| Image generation | FAL (FLUX 2 Pro) | `fal-queue-gateway.nousresearch.com` |
| Text-to-speech | OpenAI Audio | `openai-audio-gateway.nousresearch.com` |
| Browser automation | Browser Use | `browser-use-gateway.nousresearch.com` |

---

## Enabling Gateway Tools

Each tool opts in separately via `use_gateway: true` in `config.yaml`:

```yaml
web:
  use_gateway: true      # Use Firecrawl via Nous gateway (no FIRECRAWL_API_KEY needed)

image_gen:
  use_gateway: true      # Use FAL via Nous gateway (no FAL_KEY needed)

tts:
  use_gateway: true      # Use OpenAI TTS via Nous gateway (no OPENAI_API_KEY needed)

browser:
  use_gateway: true      # Use Browser Use via Nous gateway
```

When `use_gateway: true`, the gateway is preferred even if a direct API key (e.g., `FAL_KEY`) is also set.

You can also configure via `hermes model` → select Nous Portal → follow the tool setup prompts.

---

## Checking Status

```bash
hermes status
hermes tools
```

`hermes status` shows gateway availability under each tool section. `hermes tools` shows whether each tool is using the gateway or a direct key.

---

## Priority: Gateway vs Direct Keys

When both a direct API key and gateway access are available:

- `use_gateway: true` → gateway is used, direct key is ignored
- `use_gateway: false` (or not set) → direct key is used
- No direct key present → gateway is used automatically when subscription is active

---

## Environment Variable Overrides

These are advanced overrides — most users do not need them.

| Variable | Purpose |
|----------|---------|
| `TOOL_GATEWAY_USER_TOKEN` | Override the Nous access token used for gateway auth |
| `TOOL_GATEWAY_SCHEME` | Override gateway URL scheme (`http` or `https`, default `https`) |
| `TOOL_GATEWAY_DOMAIN` | Override the shared gateway domain (default `nousresearch.com`) |
| `FIRECRAWL_GATEWAY_URL` | Override the Firecrawl gateway origin (vendor-specific) |
| `FAL_QUEUE_GATEWAY_URL` | Override the FAL gateway origin |
| `OPENAI_AUDIO_GATEWAY_URL` | Override the OpenAI Audio gateway origin |
| `BROWSER_USE_GATEWAY_URL` | Override the Browser Use gateway origin |

---

## Troubleshooting

**"Tool gateway not available" / tool falls back to direct API key**
- Run `hermes model` and verify you are logged in to Nous Portal
- Ensure your Nous subscription is not free-tier
- Run `hermes status` and look for the gateway availability line

**Token expiry**
Hermes auto-refreshes the Nous access token with a 120-second skew buffer. If you see auth errors, run `hermes model` → Nous Portal → re-authenticate.

**Custom gateway endpoints**
For self-hosted or test deployments, use the `TOOL_GATEWAY_DOMAIN` env var or per-vendor overrides listed above.
